## 🏗 معماری و تکنولوژی‌ها

- **Backend:** Java 17+ / Spring Boot / REST API روی پورت `8080`
- **پایگاه داده:** SQLite (فایل دیتابیس به صورت خودکار در `source/backend/database/secondhand.db` ساخته می‌شود)
- **احراز هویت:** توکن (Bearer Token) که پس از ورود صادر می‌شود
- **Frontend:** JavaFX (OpenJFX) — بدون هیچ دسترسی مستقیم به پایگاه داده؛ همه چیز از طریق API

## ▶️ نحوه اجرا

### پیش‌نیازها
- JDK 17 یا بالاتر
- Apache Maven

### ۱) اجرای Backend

```bash
cd source/backend
mvn spring-boot:run
```

برای اطمینان از بالا بودن سرور، آدرس زیر را در مرورگر باز کنید:

```
http://localhost:8080/api/health
```

باید پیام `Backend is running` نمایش داده شود.

### ۲) اجرای Frontend (در ترمینال جداگانه)

```bash
cd source/frontend
mvn javafx:run
```

> Frontend به آدرس `http://localhost:8080` متصل می‌شود؛ حتماً ابتدا Backend را اجرا کنید.

## ✨ امکانات پیاده‌سازی‌شده

### کاربر عادی
- ثبت‌نام، ورود و خروج (با توکن)
- مشاهده لیست آگهی‌های فعال (تأییدشده)
- جستجوی کلیدواژه‌ای در عنوان و توضیحات + فیلتر بر اساس دسته‌بندی، شهر و بازه قیمت
- مشاهده جزئیات آگهی
- ثبت آگهی جدید (پس از تأیید مدیر نمایش داده می‌شود)، ویرایش و حذف آگهی‌های خود
- اعلام «فروخته شد» برای آگهی خود
- علاقه‌مندی‌ها: افزودن/حذف و مشاهده لیست در پنل کاربر
- گفتگو (چت) با فروشنده از صفحه جزئیات آگهی + لیست گفتگوها در پنل کاربر (چت روی آگهی خود شخص مجاز نیست)
- امتیازدهی ۱ تا ۵ به فروشنده همراه با نظر + نمایش میانگین امتیاز (امتیاز به خود مجاز نیست)

### مدیر (Admin)
- بررسی آگهی‌های در انتظار: تأیید یا رد
- حذف آگهی‌های نامناسب
- مشاهده لیست کاربران و مسدود/رفع مسدودی آن‌ها (کاربر مسدود امکان ورود و ارسال پیام ندارد)

### قواعد سمت سرور
- تمام اعتبارسنجی‌ها و بررسی مالکیت در Backend انجام می‌شود
- خطاها به صورت JSON با ساختار `{"message": "...", "status": ...}` برگردانده می‌شوند
